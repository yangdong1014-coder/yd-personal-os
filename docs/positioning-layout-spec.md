# 战略定位页 · 布局重构规格 v1

> 给执行 AI 的精确布局指令。本文档只管「怎么排版」，不改任何功能逻辑。
> 设计依据：本页是低频、重思考的战略校准页，使用动线是「看锚 → 对照校准 → 决定改不改目标」，自上而下一条思维链。
> 受 CLAUDE.md 第 5 条 UI 规范约束（深色 + 纸色字 + 暗橙强调 + 高信息密度 + 作战室气质）。

---

## 0 · 核心原则（排版的总纲）

```
1. 整体骨架：上下分层单列动线，不要左右分栏。
   校准是一条「看锚 → 校准 → 确认」的思维链，从上读到下就完成一次校准。
2. 视觉中心：定位锚区。它是量一切的尺子，必须第一眼最醒目，钉在最顶部。
3. 高信息密度：消灭大片空白。输入框该紧凑，不要又大又空。
4. 克制：作战室气质，不是松垮的问卷。
```

---

## 1 · 三层结构（从上到下的顺序，不可调换）

```
┌─────────────────────────────────────────────┐
│  第一层 · 定位锚（视觉中心，最醒目）          │
│  像「作战方针」一样钉在顶部                    │
├─────────────────────────────────────────────┤
│  第二层 · 新建校准（主操作区）                │
├─────────────────────────────────────────────┤
│  第三层 · 目标变更确认 + 校准历史             │
└─────────────────────────────────────────────┘
```

---

## 2 · 第一层：定位锚区（重点改造）

**问题**：当前锚区被挤在屏幕顶端，只露「北极星指标：1」一行，存在感最弱，与它的重要性完全倒挂。

**改造要求**：
- 锚区做成一张**醒目的横向卡片**，置于页面最顶部、标题正下方，进页面第一屏完整可见，不被截断。
- 卡片内用**横向网格**排列五个锚字段（第一性原理 / 身份内核 / 飞轮定义 / 当前阶段 / 北极星），每个字段是一个小格：上方小字标签，下方大字内容。
- **北极星指标**单独强调：用最大字号 + 暗橙强调色，因为它是「当前阶段唯一最重要的成功标准」，是锚中之锚。
- 卡片视觉权重要高于下面两层：可用更明显的边框或顶部暗橙色条，传达「这是基准」。
- 「编辑」按钮放卡片右上角，低调即可（锚是低频变更）。

---

## 3 · 第二层：新建校准区（消灭空旷）

**问题**：当前五个文本框又大又空，留白过度，信息密度太低。

**改造要求**：
- 五个思考字段（当前主要矛盾 / 在做但不该做 / 该做但没做 / 目标对齐审查 / 本期校准结论）改为**两列网格**排布，不要一列大框堆叠。
- 每个 textarea 默认高度收紧（约 3 行高），需要时可手动拉伸（保留 resize）。
- 「校准日期 + 校准周期」放在本区**顶部一行**，并排紧凑，不要各占一大行。
- 标签放输入框**上方**（不是左侧），紧贴输入框，减少横向留白。
- 「提交校准」按钮放本区右下角，暗橙实心。

---

## 4 · 第三层：目标变更确认 + 校准历史

**问题**：手填表单和只读列表混在一起，层次乱；灰色「暂无建议」色块过大。

**改造要求**：
- 本层用一行说明点出当前状态（「当前选中：X 校准 · pending 只读 · confirm/reject 下版开放」），保留即可。
- **手填 pending 表单**收进一个可折叠区或次要视觉层级——它是验证用的临时入口，不该和正式内容抢视觉。变更类型 / 目标名称 / 目标类型 / 变更理由紧凑排成一两行，不要每个占一大块。
- **「暂无建议」空状态**缩小：一行小灰字即可（「本条校准暂无变更建议」），不要占一个大色块。
- **校准历史**列表放最底部，每条一行紧凑展示（日期 + 周期 + 结论首句），点击加载其 pending 建议。

---

## 5 · 通用样式（贯穿三层）

```
- 全程使用系统既有深色主题变量，不要硬编码浅色。
  （本页应随系统主题切换，杨栋会用深色主题查看。）
- 卡片/区块间距统一，区块之间留呼吸感，但区块内部紧凑。
- 强调色：暗橙（#c0603a 附近），只用于：北极星、主按钮、当前选中态。
- 字体：中文 Noto Sans SC，日期/数字/版本号用 JetBrains Mono。
- 禁止：彩虹标签、渐变大背景、卡片投影堆叠、无意义动画、过度留白。
```
