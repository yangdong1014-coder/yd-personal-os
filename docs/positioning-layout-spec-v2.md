# 战略定位页 · 布局重构规格 v2（定稿）

> 给执行 AI 的精确布局指令。本文档只管「怎么排版与呈现」，不改任何功能逻辑、API、数据结构。
> 受 CLAUDE.md 第 5 条 UI 规范约束。本次是对已完成 commit-2 页面的样式/结构重构。

---

## 0 · 设计总纲（先理解，再动手）

```
1. 这是低频、重思考的战略校准页，不是高频打卡页。
2. 默认态 = 战略态势面板（进来「看战局」），不是一堆等着填的空表单。
3. 内容是一条因果链，不是三个并排板块：
   定位锚（量尺）→ 校准（拿尺量当下）→ 目标变更（校准的产出）→ 改写目标系统 → 运转结果反哺下次校准
4. 战略感来自：压倒性焦点 + 秩序 + 克制。不靠堆装饰。
5. 气质：进入指挥部，不是打开微博。
```

---

## 1 · 默认态：态势面板（核心变化）

**当前问题**：一进页面就是一排空输入框，是「填表」气质，没有战略感。

**改造**：默认态只展示「已沉淀的战略现状」，空白填写表单默认不显示。

```
进入页面默认看到（自上而下）：
┌─────────────────────────────────────────┐
│ 一层 · 定位锚（作战方针）— 已设定的锚内容  │
│   ↓ 流向标签：向下约束·拿锚当尺子审视当下   │
│ 二层 · 校准轨迹 — 最近几次校准的结论（只读） │
│   ↓ 流向标签：产出·校准结论转为目标变更建议  │
│ 三层 · 待确认的目标变更 — pending 列表      │
│   ↘ 一行小字：确认后改写目标系统→反哺下次校准 │
└─────────────────────────────────────────┘
```

**填写表单（新建校准的输入区）**：默认隐藏。在「校准轨迹」标题右侧放一个「+ 新建校准」按钮，点击后填写表单才出现（就地展开或弹出，二选一实现即可）。表单字段沿用既有：校准日期、周期、当前主要矛盾、在做但不该做、该做但没做、目标对齐审查、本期校准结论。表单内五个思考字段用 2×2 + 结论通栏的紧凑网格，禁止一列大框堆叠、禁止右半边留空白。

## 2 · 第一层：定位锚区（视觉中心）

```
- 钉在页面最顶部，顶部一条暗橙色细线（2px），标明「这是基准」。
- 北极星指标：全页最醒目元素。
    · 小标题「北极星指标」用暗橙色 + 字间距拉开
    · 北极星内容用最大字号（约 26-28px）、纸色/主文字色、500 字重
    · 下方一条 48px 暗橙短横线收尾
- 当前阶段 + 北极星绑定（见第 5 节北极星逻辑）。
- 四个锚字段（第一性原理/身份内核/飞轮定义/当前阶段）：横向四列网格，
  用 1px 细缝线分隔成仪表盘格子。每格：上方小灰标签（带图标）+ 下方内容。
- 「编辑」按钮放锚区右上角，低调（锚是低频变更）。
```

**关键修复（上一版的 bug）**：
```
✗ 锚数据为空时，绝不能整个锚区不渲染（上一版就是空数据导致锚区消失）。
✓ 锚为空时，锚区仍渲染，显示占位引导：「点击编辑，设定你的定位锚」，
  北极星位置显示「尚未设定北极星」。锚区永远是页面第一屏可见的存在。
```

---

## 3 · 第二层：校准轨迹

```
- 标题「校准轨迹」+ 时间轴图标；右侧「+ 新建校准」按钮（暗橙实心）。
- 历史校准用左竖线时间轴呈现：最新一条用暗橙竖线，旧的用灰竖线。
- 每条紧凑展示：日期 · 周期 + 主要矛盾首句 + 结论首句。
- 点击一条 → 加载它的 pending 建议到第三层（沿用既有交互）。
```

---

## 4 · 第三层：目标变更确认

```
- 标题「待确认的目标变更」+ 靶心图标。
- 每条 pending 建议：左侧 2px 暗橙竖条卡片，含变更类型+目标名、待确认徽章、理由。
- confirm/reject 按钮：本次仍按 commit-2 现状（只读展示 / 或 commit-3 已接则显示真按钮）。
  —— 本规格不改 confirm/reject 的功能逻辑，只管它的样式。
- 手填 pending 表单：收进折叠区（次要层级），不抢视觉。
- 空状态：一行小灰字「本条校准暂无变更建议」，不要大色块。
- 底部一行极淡小字说明因果闭环：确认后真实改写目标系统（既有六模块）→ 运转结果反哺下次校准。
```

## 5 · 北极星生成逻辑（手填 + 锁定阶段提醒）

```
- 北极星仍是 positioning_anchor 表的手填字段，不接 AI、不接任何自动生成。
- 锚区编辑态：「当前阶段」与「北极星」在视觉上相邻绑定。
- 北极星输入框下方加一行静态提示文字（写死）：
    「北极星 = 当前阶段唯一最重要的、可衡量的成功标准」
- 这只是文案 + 字段布局，不新增逻辑、不新增 API、不新增字段。
```

---

## 6 · 图标（线性单色 · 克制）

```
- 全部线性图标，单色（暗橙或灰），禁止彩色、禁止实心。
- 只在「区标题」和「关键语义节点」用，正文不滥用。
- 区标题：定位锚→罗盘 compass，校准轨迹→时间轴 timeline，目标变更→靶心 target。
- 锚字段：第一性原理→atom，身份内核→fingerprint，飞轮→rotate，阶段→flag。
- 节点：北极星→star，矛盾→alert-triangle，结论→flag-check，淘汰→archive。
- 用项目既有图标方案实现（base.html 侧栏已有图标体系，沿用同一套，不引入新图标库）。
```

---

## 7 · 流向连接线（让因果链可见）

```
- 锚区 ↓ 校准 之间：一条向下箭头 + 标签「向下约束 · 拿锚当尺子审视当下」。
- 校准 ↓ 变更 之间：一条向下箭头 + 标签「产出 · 校准结论转为目标变更建议」。
- 箭头暗橙、标签小灰字。让三块读起来是一条链，不是三个独立板块。
- 不要加飞轮缩略图 / 流程示意图（工具不塞说明书，保持克制）。
```

---

## 8 · 颜色与主题（死命令）

```
✗ 绝不硬编码任何颜色 hex（上一版演示用的深色 hex 仅为示意，实现严禁照抄）。
✓ 全程使用系统既有主题变量（var(--*)）。页面必须随系统主题切换（深/浅）正常显示。
- 暗橙强调色用既有主题变量，且极度克制：只用于顶部线、北极星、当前选中态、主按钮、流向箭头。
- 中文标题可适当拉开字间距（letter-spacing）以增强严肃感。
```
